Installation

Download | Dependencies
Linux | macOS | Windows




Download

GitHub
The GEM software package can be downloaded at the Large Scale GxE Github website.
https://github.com/large-scale-gxe-methods/GEM/releases.

We recommend installing the source code from the latest version of the software at the GitHub release page.

Current version is 1.1.
 
 
 
 
 

Dependencies

C/C++ Compiler
GEM is a software written in C/C++11 and so a C/C++ compiler with C++11 support is needed.
We have mostly compiled GEM with the GNU compiler collection (GCC) versions 7.5.0 ≤ x ≤ 9.3.0.
 

LAPACK and BLAS
The LAPACK(Linear Algebra PACKage) and BLAS(Basic Linear Algebra Subprograms) libraries are used to optimize matrix operations in GEM.

LAPACK/BLAS version 3.9.0 has been successfully tested.

Intel processors:
If the Math Kernal Library (MKL) is installed, we recommend linking LAPACK/BLAS through MKL for a greater performance boost. This can be done by replacing -llapack and -lblas in the makefile with
-lmkl_gf_lp64 -lmkl_sequential -lmkl_core and re-compiling the software.

It is important to compile with -lmkl_sequential since GEM already does multithreading across SNPs.

MKL versions 2019 ≤ x ≤ 2020 have been successfully tested.


AMD processors:
For AMD processors, OpenBLAS or ATLAS may be better alternatives for optimized matrix operations.
 

Boost C++ Libraries
The Boost C++ libraries are used for command-line, file management and multithreading purposes.

The following Boost libraries are required :
      1. libboost_system
      2. libboost_program_options
      3. libboost_filesystem
      4. libboost_thread

Boost versions 1.58.0 ≤ x ≤ 1.73.0 have been successfully tested.
 
 
 
 
 

Linux
After downloading the latest release of GEM, run the following lines of code:
cd GEM-[version]/
cd src/
make

Check if the installation of the software was successfully installed:
./GEM --help
