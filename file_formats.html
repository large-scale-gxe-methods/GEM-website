Input/Output Files

Genotype Files | Phenotype File | Output File | Sample File




Genotype Files

.bgen
GEM supports version 1.1, 1.2 and 1.3 binary GEN (BGEN) genotype files described here.

A .sample file should be included with the .bgen file if there is not a sample identifier block within the .bgen file. This can be done by including the --sample flag.

In GEM versions ≥ 1.1, if the .sample file is included and the .bgen file does contain a sample identifier block, GEM will prioritize the .sample file over the sample identifier block for sample identifier matching.
 
 
 
 
 

Phenotype File

.pheno
The phenotype file does not need to have the .pheno extension and can be a different type of extension (.csv, .txt, etc...). However, the appropriate delimiter of the phenotype file should be specified using the --delim parameter. The default delimiter for phenotype files in GEM is , (comma-seperated).

A phenotype file should contain a sample identifer column, along with the phenotype, covariate, interactive covariate and exposure columns. The ordering of the columns does not matter.
The first row of the phenotype file should contain the header name for each column.
 
 
 
 
 

Output File

.out
The --out flag should contain the full path and extension to where GEM output results. Otherwise, GEM will use gem.out as the default output file name and results will most likely be located in the directory where GEM was executed.



Below is a table describing the column names in the output file.

Sample File

.sample
A .sample file is required when the BGEN file does not include a sample identifier block.
Include a .sample file with the --sample option.

Formats for the .sample file should follow QCTOOLv2 format described here.
The first column in the .sample file should always be the ID column used for ID matching with the phenotype file.

In GEM versions ≥ 1.1, the .sample file will be prioritized over the sample identifier block in the BGEN file for sample identifier matching.
